- **КПЭ:** сокращение времени на подготовку документации; исключение ошибок в закупочных документах.

#### 1.2. Анализ коммерческих предложений

- **КПЭ:** автоматизация оценки КП; сокращение трудозатрат на сверку условий.

#### 1.3. ИИ-помощник для инициатора

#### 1.4. Анализ дублирующих заявок и запасов

#### 1.5. Прогнозирование потребности и заявок

### Дополнительная аналитика
БМТС — наиболее концентрированный блок по закупочным инициативам. Все инициативы замкнуты к одному контактному лицу (Макаров А.Б.), что упрощает пилотирование. «Прогнозирование потребности» — high-value, но требует предварительной оценки зрелости данных по истории заявок.

## Дополнительная аналитика

### Точки синергии между направлениями

1. **База НМД + Справочная система + Вопросно-ответный модуль** — три инициативы из разных блоков решают по сути одну задачу: быстрый поиск информации по внутренней нормативной базе. Рекомендуется рассмотреть как единый продукт «Корпоративный ИИ-поиск».

2. **Проверка комплектности заявок + Анализ дублирующих заявок + Прогнозирование потребности** — выстраиваются в цепочку «заявка → проверка → планирование». Могут быть реализованы как единый модуль в системе БМТС.

3. **Автопротокол совещаний + Мемо совещаний путем расшифровки** — дублирующие задачи из разных блоков. Рекомендуется объединить в одну инициативу «Автоматизация протоколов».

**Высокая зрелость** (данные структурированы, опыт пилотирования есть):
- Семантический поиск по ЛНА, База НМД, Проверка заявок, Анализ КП

**Средняя зрелость** (данные есть, но разрознены):
- Прогнозирование потребности, Проверка документации, Подбор подрядчиков

**Низкая зрелость** (требуется сбор и структурирование данных):
- Предиктивные модели (выходы из строя, загруженность ПБУ), Моделирование грузооборота

### Рекомендации по старту MVP-волн

**Волна 1 (быстрые победы, 1–3 месяца):**
