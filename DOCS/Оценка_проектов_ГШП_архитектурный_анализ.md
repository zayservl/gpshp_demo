## Ключевые наблюдения архитектора ИИ-решений

### 1. Занижение сложности заказчиком
Многие проекты помечены как "Низкий риск / 4", но реально требуют **5-6 баллов**:
- Семантический поиск по ЛНА
- ИИ-трансформация документов (юридические заключения → мемо)
- Генерация ТЗ, оптимизация документации

### 2. Стоимость заказчика занижена
Особенно для NLP/LLM-проектов на больших данных:
- Реальность: **10-15 млн**, а не 5-10
- Проекты с терабайтами данных требуют серьёзной RAG-архитектуры

### 3. Не ИИ vs ИИ
Часть проектов помечена "Не ИИ" (RPA), но при желании можно добавить ИИ-компонент:
- Проверка договоров → ИИ-анализ текста
- Сравнение предложений → fuzzy matching с ИИ
- Моделирование вахт → ИИ-предиктивное планирование

### 4. Критичные системы (Высокий риск / 10)
Требуют отдельного обоснования и отдельного бюджета:
- ИИ-видеоаналитика на производстве
- Предиктивная безопасность
- АСУ флотом
- Юридический Due Diligence

### 5. Рекомендации по приоритизации

**Быстрые победы (3-5 млн, сложность 2-4):**
- VBA → ИИ (автозаполнение форм)
- RPA для проверки КА
- Автоматизация сводок
- Проверка текстов на опечатки

**Средние проекты (5-10 млн, сложность 5-7):**
- ИИ-помощник для инициатора/закупщика
- Семантический поиск
- Автопротокол совещаний
- Классификация замечаний

**Сложные проекты (10-20 млн, сложность 8-9):**
- Предиктивное ТО
- ИИ-видеоаналитика
- Системы технического надзора
- RAG на больших объёмах

**Критичные проекты (20+ млн, сложность 9-10):**
- АСУ флотом
- Предиктивная безопасность
- Due Diligence
- Промышленная видеоаналитика
