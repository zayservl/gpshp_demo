| Атрибут | Значение |

#### 3.1.1 Назначение

Автоматическое формирование ежедневных сводок (отчётов) на основе данных из информационных систем предприятия. Цель — исключить ручной труд по сбору и структурированию данных для подготовки управленческих отчётов.

#### 3.1.2 Функциональность

**Основные функции:**
1. Сбор данных из источников (моковые данные):
   - Топливный отчёт (остатки, расход, приход)
   - Отчёт о движении судов
   - Отчёт о выполнении работ
   - Отчёт о нештатных ситуациях

2. Обработка и агрегация данных:
   - Фильтрация по периоду (сутки, неделя, месяц)
   - Группировка по судам/подразделениям
   - Расчёт ключевых показателей

3. Формирование сводки:
   - Шаблонная структура документа
   - Автозаполнение полей
   - Экспорт в PDF/Word

4. Уведомление:
   - Отправка сводки ответственным
   - Логирование отправки

#### 3.2.1 Назначение

Автоматизация процесса сбора, обработки и контроля исполнения предложений по улучшению деятельности предприятия от сотрудников. Включает регистрацию предложений, классификацию, назначение ответственных и мониторинг исполнения.

#### 3.2.2 Функциональность

**Основные функции:**
1. Регистрация предложения:
   - Форма подачи (web-интерфейс)
   - Автоматическая нумерация
   - Привязка к категории

2. Классификация предложений:
   - Категория (техническая, организационная, экономическая, безопасность)
   - Приоритет (высокий, средний, низкий)
   - Направление (производство, логистика, управление)

3. Маршрутизация:
   - Назначение ответственного подразделения
   - Определение срока рассмотрения

4. Мониторинг исполнения:
   - Статусы (новое, на рассмотрении, в работе, выполнено, отклонено)
   - Уведомления о просрочке

5. Отчётность:
   - Реестр предложений
   - Статистика по категориям
   - Сроки исполнения
