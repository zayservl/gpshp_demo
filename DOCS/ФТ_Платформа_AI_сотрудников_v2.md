**Версия:** 2.0

**Статус:** MVP для демонстрации заказчику

---

## 1. Общее описание

**Платформа AI-сотрудников** — система интеллектуальных цифровых работников, каждый из которых является оркестратором агентов. AI-сотрудник самостоятельно анализирует запрос, выбирает инструменты, составляет план и согласовывает его с пользователем.

### Направления AI-сотрудников

| Направление | AI-сотрудник | Ключевые функции |
|---|---|---|
| Делопроизводство | **Анна Делопроизводитель** | Проверка заявок, анализ документов, контроль комплектности |
| Юриспруденция | **Александр Юрист** | Анализ договоров, консультации по документам, оценка рисков |
| Закупки | **Николай Закупщик** | Анализ КП, сравнение предложений, оценка поставщиков |
| Ассистент | **Ассистент ГПШ** | Подготовка ответов, работа с обращениями, генерация контента |

## 2. Структура интерфейса

### 2.1 Вкладка «Сотрудники»

**Описание:** Список активных AI-сотрудников

**Элементы:**
- Карточка сотрудника:
  - Название и аватар
  - Направление (деlopроизводство / юриспруденция / закупки / ассистент)
  - Статус: «активен»
  - Список доступных инструментов
  - Кнопка «Написать» → открытие чата

**Критерии приёмки:**
- Отображаются только «активные» сотрудники (по одному на направление)
- Виден полный список инструментов сотрудника
- Клик по сотруднику открывает диалог

### 2.2 Вкладка «Dashboard»

**Описание:** Панель мониторинга системы для понимания происходящего

**Элементы:**
- **Выполнено задач** — общее число + разбивка (успешно / неуспешно)
- **Сценарии в работе** — текущие активные задачи с статусом
- **График запусков** — активность сотрудников по времени (гистограмма)
- **Метрики** — среднее время выполнения, загрузка сотрудников

**Критерии приёмки:**
- Данные обновляются в реальном времени
- График отображает активность за последние 24 часа

### 2.3 Вкладка «Данные»

**Описание:** Хранилище документов и данных в работе

**Элементы:**
- Список документов с метаданными (название, тип, дата, статус)
- Превью содержимого при клике
- Фильтры: по сотруднику, по типу, по статусу
- История изменений документа

**Критерии приёмки:**
- Пользователь может «протыкать» любой документ
- Отображается полное содержимое документа

## 3. AI-сотрудник как оркестратор

### 3.1 Приём запроса

**Описание:** Пользователь отправляет запрос на естественном языке через чат

**Требования:**
- Поддержка русского языка
- Понимание намерений («проверить», «сравнить», «подготовить»)
- Извлечение сущностей: номера документов, даты, суммы, имена

**Примеры запросов:**
- «Проверь заявку №124»
- «Проанализируй договор Д-2026-001 на риски»
- «Сравни КП от ТехноПоставка и ИП Сидоров»
- «Подготовь ответ на обращение №45-2026»

### 3.2 Анализ и планирование

**Описание:** AI-сотрудник анализирует запрос и составляет план действий

**Процесс:**
1. Распознавание намерения
2. Извлечение ключевой информации
3. Выбор инструментов из доступных
4. Определение необходимых данных
5. Формирование плана действий

**Требования:**
- AI выбирает инструменты из списка доступных сотруднику
- План включает название шага, инструмент, источник данных
- Указываются документы, которые будут использованы

### 3.3 Согласование плана

**Описание:** План действий отправляется на согласование пользователю

**Элементы плана:**
```
📋 ПЛАН ДЕЙСТВИЙ

1. [Название шага]
   → Инструмент: <имя>
   → Источник: <документ/система>

2. [Название шага]
   → ...

Документы:
- <список документов с названиями>

[✅ Выполнить] [❌ Отмена]
```

**Критерии приёмки:**
- План показан до начала выполнения
- Пользователь может подтвердить или отклонить
- При отказе — возврат в чат

### 3.4 Выполнение

**Описание:** После согласования AI-сотрудник выполняет план

**Требования:**
- Последовательное выполнение шагов
- Визуализация графа выполнения (текущий шаг подсвечен)
- Логирование каждого шага
- Отображение прогресса

**Элементы графа:**
- Узлы — шаги плана
- Ребра — зависимости между шагами
- Состояния узлов: ожидание, выполнение, завершено, ошибка

### 3.5 Результат

**Описание:** По завершении показывается результат

**Элементы:**
- Сводка выполненных действий
- Ссылки на созданные документы
- Предложение следующих действий

## 4. Чат с AI-сотрудником

### 4.1 Интерфейс чата

**Элементы:**
- Поле ввода сообщения
- История сообщений (пользователь + AI)
- Индикатор «AI думает...»
- Кнопка «Логи ИИ»

### 4.2 Логи ИИ

**Описание:** Внутренние промпты и рассуждения модели

**Элементы:**
- Системный промпт сотрудника
- Промежуточные рассуждения (reasoning)
- Выбранные инструменты и данные
- Формирование ответа

**Критерии приёмки:**
- Доступно по кнопке в чате
- Показывает полный контекст принятия решений

## 6. Граф выполнения

### 6.1 Визуализация

**Элементы:**
- React Flow граф
- Узлы: прямоугольники с названием шага
- Ребра: стрелки между узлами
- Анимация: подсветка текущего узла
- Клик по узлу: детали (входные данные, результат)

### 6.2 Состояния узлов

| Состояние | Визуализация |
|---|---|
| ожидание | серый |
| выполнение | синий (пульсация) |
| завершено | зелёный |
| ошибка | красный |

## 8. Критерии приёмки MVP

| Требование | Критерий |
|---|---|
| Вкладка «Сотрудники» | Показаны 4 активных сотрудника с инструментами |
| Вкладка «Dashboard» | Отображаются метрики и графики |
| Вкладка «Данные» | Список документов с возможностью просмотра |
| AI-сотрудник понимает запрос | Достаточно 3-4 тестовых запросов |
| Согласование плана | План показан до выполнения |
| Граф выполнения | Визуализация с анимацией |
| Логи ИИ | Доступны в чате |
| Чат работает | Обмен сообщениями функционирует |
