# Демонстрация платформы AI-сотрудников ГШП

## Концепция

**AI-сотрудник** — это интеллектуальный оркестратор агентов, который:
1. Понимает запрос пользователя на естественном языке
2. Анализирует контекст и извлекает ключевую информацию
3. Выбирает необходимые инструменты из доступных
4. Составляет план действий с указанием документов
5. **Отправляет план на согласование** пользователю
6. После подтверждения выполняет задачу до конца

## Структура интерфейса

### Вкладка «Сотрудники»
Показывает активных AI-сотрудников — по одному для каждого направления:

| Направление | AI-сотрудник | Назначение |
|---|---|---|
| Делопроизводство | **Анна Делопроизводитель** | Проверка документов, анализ заявок, контроль комплектности |
| Юриспруденция | **Александр Юрист** | Анализ договоров, консультации по документам, проверка рисков |
| Закупки | **Николай Закупщик** | Анализ КП, сравнение предложений, оценка поставщиков |
| Ассистент | **Ассистент ГПШ** | Подготовка ответов, работа с обращениями, генерация контента |

Каждый сотрудник показан с:
- Названием и ролью
- Списком доступных инструментов
- Кнопкой «Написать» для начала диалога

### Вкладка «Dashboard»
Панель мониторинга системы:

- **Выполнено задач**: общее число, успешно / неуспешно
- **Сценарии в работе**: текущие активные задачи
- **График запусков**: активность сотрудников по времени
- **Метрики**: среднее время выполнения, загрузка сотрудников

### Вкладка «Данные»
Хранилище данных в работе:
- Документы, которые использует AI-сотрудник
- Промежуточные результаты
- Сгенерированные отчёты
- Пользователь может «протыкать» — посмотреть содержимое любого документа

## Как работает AI-сотрудник (оркестратор)

### 1. Приём запроса
Пользователь пишет запрос на естественном языке:
> «Проверь заявку на комплектность»

### 2. Анализ и планирование
AI-сотрудник:
- Распознаёт намерение
- Извлекает ключевые данные (номер заявки, дата)
- Выбирает инструменты из доступных
- Формирует план действий

### 3. Согласование плана
Пользователю показывается план:
```
📋 ПЛАН ДЕЙСТВИЙ

1. Загрузить заявку №124
   → Источник: База заявок ГШП
   
2. Проверить обязательные поля
   → Инструмент: document_checker
   
3. Сверить с перечнем документов
   → Источник: Регламент проверки.docx
   
4. Сформировать отчёт об ошибках
   → Результат: Отчёт о комплектности.docx

Документы:
- Заявка_124.pdf
- Регламент_проверки.pdf
- Перечень_документов.xlsx

Выполнить? ✅ Да / ❌ Нет
```

### 4. Выполнение
После согласия AI-сотрудник:
- Последовательно выполняет шаги
- Показывает прогресс в графе
- Логирует каждый шаг
- По завершении показывает результат

### 5. Результат
- Сводка выполненных действий
- Ссылки на созданные документы
- Предложение следующих действий

## Сценарии демонстрации

### Сценарий 1: Проверка заявки на комплектность
**Сотрудник:** Анна Делопроизводитель  
**Запрос:** «Проверь заявку №124»

**Процесс:**
1. AI понимает: нужно проверить документ
2. Выбирает инструменты: document_checker, text_analyzer
3. Запрашивает данные: заявка из базы
4. Формирует план → показывает согласовние
5. После подтверждения — проверяет
6. Результат: отчёт с ошибками

### Сценарий 2: Анализ договора на риски
**Сотрудник:** Александр Юрист  
**Запрос:** «Проанализируй договор Д-2026-001 на риски»

**Процесс:**
1. AI понимает: нужен анализ договора
2. Выбирает инструменты: contract_analyzer, risk_assessment, legal_search
3. Запрашивает договор и регламенты
4. Показывает план с документами
5. Выполняет анализ по 44-ФЗ
6. Результат: заключение с рисками

### Сценарий 3: Сравнение коммерческих предложений
**Сотрудник:** Николай Закупщик  
**Запрос:** «Сравни КП от ТехноПоставка и ИП Сидоров»

**Процесс:**
1. AI: анализ КП
2. Выбирает: kp_analyzer, price_comparison
3. Показывает план с КП-001, КП-002
4. Сравнивает по критериям
5. Результат: таблица сравнения + рекомендация

### Сценарий 4: Подготовка ответа на обращение
**Сотрудник:** Ассистент ГПШ  
**Запрос:** «Подготовь ответ на обращение гражданина №45-2026»

**Процесс:**
1. AI: генерация ответа
2. Выбирает: semantic_search, text_generator
3. Ищет шаблон, классифицирует обращение
4. Показывает план
5. Генерирует проект письма
6. Результат: проект ответа

## Что видит пользователь

### Чат с AI-сотрудником
- История диалога
- Запросы и ответы
- Кнопка «Показать логи ИИ» — внутренние промпты и рассуждения модели

### Граф выполнения
- Узлы — шаги плана
- Анимация переходов
- При клике на узел — содержимое (входные данные, результат шага)

### Вкладка «Данные»
- Список всех документов в работе
- Превью содержимого
- История изменений
